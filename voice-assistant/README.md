# Voice Assistant

A voice-first AI assistant built on three layers:

```
┌─────────────────────────────────────────────────────────────┐
│  Browser (React + MediaRecorder)                            │
│  Press the mic → record → POST audio → hear spoken reply    │
└──────────────────────────┬──────────────────────────────────┘
                           │ audio blob
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  Handler (Hono on Cloudflare)                               │
│  STT → LLM (structured JSON) → dispatch actions → TTS       │
│  ┌────────────┐  ┌──────────┐  ┌──────────┐  ┌────────────┐ │
│  │ ElevenLabs │→ │ Claude / │→ │  Gmail   │→ │ ElevenLabs │ │
│  │   STT      │  │ any LLM  │  │ Calendar │  │   TTS      │ │
│  └────────────┘  └──────────┘  └──────────┘  └────────────┘ │
└─────────────────────────────────────────────────────────────┘
                           │
                           ▼
                 Sauna proxy (sauna.local)
                 ElevenLabs + LLM — no keys needed
```

## What it does

1. You tap the mic and speak a request
2. Your browser records audio (WebM / Opus) and sends it to the handler
3. The handler transcribes via **ElevenLabs Scribe**
4. The handler asks an **LLM (Claude)** to interpret the request — the LLM returns a structured JSON plan with a short spoken reply and zero or more actions
5. The handler dispatches each action (Gmail draft, Google Calendar event)
6. The handler streams the spoken reply back as MP3 via **ElevenLabs Turbo/Flash TTS**
7. Your browser plays the audio and renders the conversation transcript + action chips

## Stack

- **Voice I/O**: [ElevenLabs](https://elevenlabs.io) via Sauna proxy (no account needed)
- **Reasoning**: any LLM via Sauna proxy (no API key needed) — defaults to `balanced`
- **Memory/Orchestration**: Sauna App with SQLite + Drizzle ORM
- **Actions**: Gmail drafts + Google Calendar events (own OAuth connections)
- **Frontend**: React 18, inline SVGs (no icon libraries)
- **Backend**: Hono on Cloudflare

## Deploy

This app lives inside Sauna. From a Sauna session:

```bash
# 1. The folder is already created at apps/voice-assistant/

# 2. Generate the migration from src/schema.ts
app_generate_migration { project: "voice-assistant", name: "init" }

# 3. Deploy (pin Gmail + Calendar as stored connections, Docs + Sheets as Pipedream accounts)
app_deploy {
  slug: "voice-assistant",
  connections: [
    { id: "<gmail-stored-connection-id>", name: "Gmail" },
    { id: "<calendar-stored-connection-id>", name: "Google Calendar" }
  ],
  accounts: [
    { id: "<google-docs-pipedream-apn-id>", name: "Google Docs", app: "google_docs" },
    { id: "<google-sheets-pipedream-apn-id>", name: "Google Sheets", app: "google_sheets" }
  ]
}
```

The deploy returns a URL like `https://voice-assistant-<8chars>.sauna.new/`.

## How to test

Open the URL in a browser. The browser will ask for microphone permission.

Try phrases like:

- *"Read my last 5 emails"* — opens your inbox
- *"What's in my inbox from alice?"* — Gmail search filter
- *"Draft an email to alice@example.com about the Q3 review"*
- *"Schedule a 30-minute meeting tomorrow at 2pm with bob@acme.com"*
- *"Create a Google Doc called Q3 Plan with the bullets we discussed"*
- *"Create a sheet called Monthly Expenses with columns Date, Category, Amount and add a few rows"*
- *"Both — email bob that I'm running late, and move our 3pm to 4pm"*

The assistant will speak its reply out loud and show chips for each action (✉️ Draft, 📧 Read N, 📅 Event, 📄 Doc, 📊 Sheet) with ✓/✗ status. Read actions list subjects + senders inline. Docs/Sheets actions include clickable "open" links. Email actions land as Gmail **drafts** (not sent) by default — safer for hands-free demos.

## The spoken-word system prompt

The single biggest gotcha for voice-first apps: LLMs default to markdown output. TTS will literally read "*asterisk asterisk bold asterisk*". The system prompt in `src/llm.ts` enforces:

- No markdown, no bullets, no lists, no emojis, no URLs
- Short spoken sentences
- One-or-two-sentence confirmations

Plus a strict JSON shape for actions, parsed defensively:

```json
{
  "spoken_reply": "I've drafted that email and scheduled the meeting.",
  "actions": [
    {"type": "gmail_draft", "to": "...", "subject": "...", "body": "..."},
    {"type": "calendar_event", "title": "...", "start": "...", "end": "...", "attendees": ["..."]}
  ]
}
```

## Adding a new action type

1. Add the variant to `Action` in `src/llm.ts` and update the system prompt
2. Add the dispatch branch in `src/actions.ts` — set the right `X-Sauna-Connection-Id` per request when multiple pinned connections share a host
3. Update the chip rendering in `src/client.tsx` so the UI shows it

## Latency notes

- **TTS streaming** is the biggest lever: the handler pipes the upstream MP3 body straight back to the browser (binary, no SSE framing to parse)
- **One round-trip per turn**: STT → LLM → dispatch all happen inside `/api/turn` so the browser doesn't pay three sequential trips
- **LLM tier**: `balanced` is the cheapest tier that handles structured JSON extraction reliably
- **402 (owner out of credits)**: surfaces "temporarily unavailable" in the UI, no retry loop, no half-persisted state
